Desafio de Deploy: Colocando o Cofre Digital Online!

Objetivos da Atividade

  1. Publicar o projeto do cofre digital em um site acessível pela internet.
  2. Aprender a usar o GitHub para versionamento de código.
  3. Integrar o GitHub com o Netlify para deploy automático.
  4. Praticar documentação de projetos com README.md.

Instruções Gerais

1. Configuração no GitHub

  • Crie um repositório público no GitHub chamado cofre-digital.
  • Faça upload de todos os arquivos do projeto (HTML, CSS, JavaScript e imagens).
  • Certifique-se de que o arquivo principal seja index.html.

Dica:

git add .
git commit -m "Primeiro commit: projeto do cofre digital"
git push origin main

2. Deploy no Netlify

  1. Acesse Netlify e faça login com sua conta do GitHub.
  2. Clique em "Add new site" > "Import an existing project".
  3. Selecione o repositório cofre-digital e autorize a conexão.
  4. Deixe as configurações padrão e clique em "Deploy site".
  5. Aguarde o deploy finalizar e anote a URL gerada (ex: https://meu-cofre.netlify.app).

3. Documentação no README.md

Crie um arquivo README.md na raiz do repositório com:

  • Título do projeto
  • Breve descrição do cofre digital
  • URL do Netlify
  • Tecnologias usadas (ex: HTML, CSS, JavaScript)
  • Instruções para executar localmente (opcional)

Exemplo:

# Cofre Digital

Projeto de interface segura para armazenar senhas e mensagens.

**Acesso Online**: [https://meu-cofre.netlify.app](https://meu-cofre.netlify.app)

## Tecnologias
- HTML5
- CSS3
- JavaScript

## Como Executar
Clone o repositório e abra `index.html` no navegador.

O Que Será Avaliado?

Critério Descrição
Funcionalidade Online O site deve estar acessível via URL do Netlify sem erros.
Documentação README.md claro e completo com todas as informações solicitadas.
Integração Contínua Deploy automático configurado corretamente no Netlify.
Organização do Código Estrutura de pastas e arquivos lógica no GitHub.

Dicas Extras

  • Teste o site no Netlify após o deploy.
  • Se houver erros, verifique o log de deploy no Netlify.
  • Atualize o repositório se precisar corrigir algo (o Netlify reimplanta automaticamente).

Perguntas Frequentes (FAQ)

  1. "Meu deploy falhou. O que fazer?" Verifique se o index.html está na raiz do repositório e se todos os arquivos foram commitados.

  2. "Posso usar outro serviço além do Netlify?" Não para esta atividade. O objetivo é aprender a usar a integração GitHub + Netlify.

  3. "Preciso comprimir os arquivos?" Não. O Netlify lida com arquivos brutos. Mantenha a estrutura original.
